SMP support

CMRX kernel supports SMP-aware configuration. If SMP operation is enabled via CMRX_ARCH_SMP_SUPPORTED then there are few changes in CMRX behavior:

  • Kernel is started per-core. The prototype of function os_start changes. It now accepts an argument, which denotes, for which core the kernel is being started. This function can't be called from within thread and it won't return. The order in which cores are started is not important, just mind the order in which auto-started threads will begin execution.
  • Thread auto-starting gets one additional argument which identifies core on which thread is meant to be started.
  • Additional API os_thread_migrate is provided which allows to migrate thread between cores.

In SMP arrangement, kernel maintains run queue for each core on which CMRX has been booted. This means that you may have a multicore machine with e.g. 4 cores, run some non-managed code there (such as radio firmware) and boot CMRX on three remaining cores. Then you can divide workload between those three cores.

Signal distribution initiated both from thread and ISR currently does not propagate cross-core reliably.
